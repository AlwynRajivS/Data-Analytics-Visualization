Foundation
What is Cluster Analysis?

Cluster analysis groups objects so that members of the same cluster are more similar to each other than to those in other groups. It is a core unsupervised learning technique — no labelled training data required.

🛒 Real World
E-Commerce Customer Segmentation

Amazon clusters millions of shoppers into groups — budget hunters, brand loyalists, impulse buyers — without any human labelling each customer. The algorithm discovers these patterns purely from purchase history and browsing data.

  • Budget hunters → receive discount coupons & flash sale alerts
  • Brand loyalists → receive premium new arrival launches
  • Impulse buyers → shown high-margin trending items
💡
Key InsightClustering is unsupervised — unlike classification (spam/not spam), there are no predefined labels. The algorithm discovers natural groupings entirely from the data.
Mathematics
Distance Measures

Every clustering algorithm depends on a distance or similarity function. Choosing the right measure is critical to getting meaningful clusters.

Euclidean Distance — Most Common
d(p,q) = √[ (x₁−x₂)² + (y₁−y₂)² ]
p, qtwo data points
x₁, x₂feature 1 values
y₁, y₂feature 2 values
Manhattan Distance — City Block
d(p,q) = |x₁−x₂| + |y₁−y₂|
| · |absolute value
grid pathlike city blocks
🏠 Worked Example
Housing Prices — Are Two Houses Similar?

House A: 1200 sq ft, ₹50L  |  House B: 1800 sq ft, ₹80L

d = √[ (1200−1800)² + (50−80)² ] = √[ 360000 + 900 ] = √360900 ≈ 600.75

⚠️ Units differ wildly — must normalise features before computing distance or sq ft dominates entirely.

Data Types
Types of Variables

Different variable types require different distance measures. Choosing wrong is a common cause of poor results.

📏

Interval-Scaled

Continuous values with equal spacing.
Examples: salary, temperature, height in cm.

🔢

Binary

Exactly two values: 0 or 1.
Examples: gender (M/F), has_credit_card, churned.

🏷️

Nominal

Named categories, no order.
Examples: religion, city, payment_method.

📊

Ordinal

Ordered categories, unequal intervals.
Examples: military rank, satisfaction (Low/Med/High).

📐

Ratio-Scaled

Numeric with a true zero point.
Examples: income (₹0 possible), population growth.

🔀

Mixed Types

Combine several types. Requires Gower's distance.
Example: patient records (age + gender + cost).

K

K-Means Clustering

Partitions n observations into K clusters. Each point belongs to the cluster with the nearest mean (centroid). Minimises Within-Cluster Sum of Squares (WCSS).

Time: O(n·k·t·d)
Objective — Minimise Within-Cluster Sum of Squares
J = Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²
Jtotal inertia (lower = better clusters)
Cₖset of points in cluster k
xᵢdata point i
μₖcentroid of cluster k
‖·‖²squared Euclidean distance
Centroid Update Rule (E-Step → M-Step)
μₖ = (1 / |Cₖ|) × Σxᵢ   for all xᵢ ∈ Cₖ
|Cₖ|number of points in cluster k
Σxᵢsum of all point coordinates
μₖnew centroid = mean of cluster
1

Choose K & Initialise Centroids

Pick K data points randomly (or use K-Means++ which spreads centroids apart to reduce bad initialisation).

2

Assign Each Point to Nearest Centroid

Compute Euclidean distance from every point to every centroid. Assign point to the closest cluster.

3

Recompute Centroids

For each cluster, calculate the mean (μₖ) of all assigned points. This is the new centroid.

4

Repeat Until Convergence

Repeat steps 2–3 until centroids move less than threshold ε, or max iterations reached.

🏥 Real World
Hospital Patient Risk Grouping (K=3)

Features: age, BMI, blood pressure, cholesterol. K-Means finds 3 natural risk levels:

  • Cluster 1 — Low Risk: Age 20–35, normal BMI → routine check-ups
  • Cluster 2 — Moderate Risk: Age 36–55, slightly elevated → quarterly monitoring
  • Cluster 3 — High Risk: Age 55+, high BMI, elevated BP → immediate intervention
⚠️
LimitationsSensitive to initial centroids (run multiple times). Assumes spherical, equal-sized clusters. Outliers drag centroids. Must specify K in advance — use the Elbow Method to find optimal K.
Elbow Method — Finding Optimal K
For each k ∈ [2, max_k]:
  fit KMeans(k) → record WCSS[k]
plot k vs WCSS → pick "elbow" point
H

Hierarchical Clustering

Builds a tree (dendrogram) of nested clusters. No need to specify K — cut the tree at any level. Agglomerative (bottom-up) is most widely used.

Agglomerative: O(n² log n)
⬆️

Agglomerative (Bottom-Up)

Start: every point = own cluster.
Merge the closest pair repeatedly.
End: one cluster. Most used.

⬇️

Divisive (Top-Down)

Start: all points in one cluster.
Split the most heterogeneous cluster.
End: each point alone. O(2ⁿ) — rare.

Four Linkage Criteria
Single:  d(A,B) = min{ d(a,b) : a∈A, b∈B }

Complete: d(A,B) = max{ d(a,b) : a∈A, b∈B }

Average:  d(A,B) = (1/|A||B|) Σₐ Σᵦ d(a,b)

Centroid: d(A,B) = ‖μ_A − μ_B‖
A, Btwo clusters being compared
a, bindividual points within clusters
|A|, |B|cluster sizes
μ_A, μ_Bcluster centroids
🧬 Real World
Gene Expression Analysis

Biologists cluster thousands of genes by expression patterns across conditions (healthy vs cancer). The dendrogram reveals co-expressed genes that likely share regulatory pathways. Scientists cut the tree at the level giving biologically meaningful groups (e.g., 5 clusters = 5 gene families).

📊
Reading a DendrogramThe height of each merge = the distance at which two clusters joined. A large gap in merge heights signals a natural cluster boundary — cut the tree there.
D

DBSCAN — Density-Based Spatial Clustering

Groups densely-packed points into clusters of any shape. Points in sparse regions are labelled as noise/outliers — not forced into clusters.

Time: O(n log n) with spatial index
⭐

Core Point

Has ≥ MinPts neighbours within radius ε. Forms the dense interior.

🔵

Border Point

Within ε of a core point, but has < MinPts own neighbours. Lives on cluster edge.

❌

Noise (Outlier)

Neither core nor border. Not density-reachable from any core point.

ε-Neighbourhood & Core Point Definitions
Nε(p) = { q ∈ D : d(p,q) ≤ ε }

p is a Core Point ⟺ |Nε(p)| ≥ MinPts
Nε(p)ε-neighbourhood of point p
Dfull dataset
εradius (user-defined)
MinPtsdensity threshold
1

Pick an Unvisited Point

Select any unvisited point from the dataset.

2

Find ε-Neighbourhood

Count all points within radius ε. If fewer than MinPts → mark as noise (temporarily).

3

Expand Cluster from Core Point

If ≥ MinPts neighbours found, start a new cluster. Recursively add all density-reachable points.

4

Repeat Until All Points Visited

Points not belonging to any cluster remain noise. Clusters can be any shape.

🌍 Real World
Geospatial Crime Hotspot Detection — Delhi

GPS coordinates of reported crimes with ε=500m, MinPts=10. DBSCAN finds irregularly shaped hotspots along busy bazaar streets or metro corridors — zones K-Means would split incorrectly.

  • Result: 7 high-density crime zones + 43 isolated incidents (noise)
  • Isolated incidents are flagged as anomalies, not forced into a cluster
✅
DBSCAN AdvantagesNo need to specify K. Finds clusters of arbitrary shape. Explicit outlier labelling. Works excellently for geospatial and sensor data.

🎯

Use K-Means When…

  • You know (or can estimate) K
  • Clusters are roughly spherical
  • Dataset is large (>100K points)
  • Speed is a priority
  • Features are numeric & normalised

Netflix — clustering 200M users into viewing preference segments for recommendations.

🌳

Use Hierarchical When…

  • Need a visual dendrogram
  • K is completely unknown
  • Small–medium dataset (<10K)
  • Want nested cluster structure
  • Interpretability is critical

Genomics — grouping 500 species by DNA similarity to build evolutionary trees.

📍

Use DBSCAN When…

  • Clusters have irregular shapes
  • Outlier detection is needed
  • K is unknown
  • Data has varying densities
  • Geospatial or sensor data

Traffic — accident hotspot mapping across city road networks.

💡
Pro Tip — Elbow Method for K-MeansPlot WCSS (inertia) vs K. The "elbow" — where the curve bends sharply — suggests the optimal K. Beyond this point, adding more clusters gives diminishing returns on variance explained.
