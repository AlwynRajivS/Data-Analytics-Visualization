Auto-Regressive (AR) Model — Deep Dive

Core Idea

An AR model predicts the current value of a series using a weighted linear combination of its own past observations. The model is "auto" because it regresses on itself.

When to Apply

Use AR when past values correlate with future values — when momentum or inertia exists. Ideal for stock prices, temperature, electricity demand, and economic indicators.

Core Requirement

The series must be stationary: constant mean and variance over time. Non-stationary data must be differenced first, which leads to the ARIMA model.

AR(p) — General Formula
$$Y_t = c + \phi_1 Y_{t-1} + \phi_2 Y_{t-2} + \cdots + \phi_p Y_{t-p} + \varepsilon_t$$
$Y_t$ = value at time $t$  |  $c$ = constant (intercept)  |  $\phi_i$ = AR coefficients  |  $p$ = model order  |  $\varepsilon_t \sim WN(0,\sigma^2)$ = white noise
Stationarity Check: Always apply the Augmented Dickey-Fuller (ADF) Test before fitting AR. If p-value > 0.05, the series is non-stationary — difference it (d≥1) and re-test. The PACF plot tells you the AR order p.
🧠
Theory Deep Dive: What Do AR Coefficients Mean?
1

Stationarity — Why It Matters

AR models assume the statistical properties of the series don't change over time. A stationary series has a constant mean, constant variance, and autocovariance that depends only on lag, not on time.

Test: ADF Test H₀: "Series has a unit root (non-stationary)". Reject H₀ (p < 0.05) → stationary. Also check KPSS test for confirmation. Visually, a flat rolling mean = stationary.
2

Choosing Order p — The PACF Method

The Partial Autocorrelation Function (PACF) measures the correlation between $Y_t$ and $Y_{t-k}$ after removing the effects of intermediate lags. The lag where PACF cuts off (drops below the 95% confidence band) is your AR order p.

PACF Rule for AR Order: • PACF significant at lags 1,2 only → AR(2) • PACF significant at lags 1,2,3 only → AR(3) • If PACF decays gradually → MA model may fit better • ACF should tail off (decay) for a pure AR process
3

Interpreting Coefficients φᵢ

Each φᵢ represents the direct effect of lag i on the current value, holding all other lags constant. Think of it as a partial regression coefficient.

Stability condition: For AR(1), the model is stationary only if |φ₁| < 1. For AR(p), all roots of the characteristic polynomial must lie outside the unit circle. If φ₁ ≈ 1, the series behaves like a random walk.
4

Estimation: Yule-Walker & OLS

AR coefficients are estimated via Yule-Walker equations (method of moments) or Ordinary Least Squares (OLS). OLS treats lagged values as regressors and minimises the sum of squared residuals.

OLS Setup for AR(2): Y = [Y₃, Y₄, ..., Yₙ]ᵀ X = [[Y₂, Y₁], ← lags for t=3 [Y₃, Y₂], ← lags for t=4 ... [Yₙ₋₁, Yₙ₋₂]] ← lags for t=n β̂ = (XᵀX)⁻¹Xᵀy → [φ̂₁, φ̂₂]
5

Model Selection: AIC vs BIC

When choosing between AR(1), AR(2), ..., AR(p), use information criteria to balance goodness of fit vs complexity.

AIC = 2k − 2ln(L̂) [Akaike Information Criterion] BIC = k·ln(n) − 2ln(L̂) [Bayesian Information Criterion] k = number of parameters | L̂ = maximised likelihood → Choose the model with LOWEST AIC or BIC → BIC penalises complexity more → selects smaller p → Both disagreeing → prefer parsimony (smaller model)

Worked Example: Stock Price Prediction with AR(2)
Problem Setup: A stock has closing prices (₹) over 5 days: 105, 108, 112, 110, 114. We fit AR(2) using estimated coefficients φ₁ = 0.65 and φ₂ = 0.30 (intercept c ≈ 0). Predict Day 6 price.
1

Verify Stationarity (ADF Test)

Check if the closing price series fluctuates around a constant mean with no clear trend. For 5 days: 105→108→112→110→114, the mean ≈ 109.8. No explosive trend — assume approximately stationary for this short window. ADF would be applied on longer real series.

2

Identify Lag Values for AR(2)

We need the two most recent past observations as inputs to the model.

To predict Day 6: Y_{t-1} = Day 5 closing price = ₹114 Y_{t-2} = Day 4 closing price = ₹110
3

Apply the AR(2) Formula

Substitute lag values and coefficients: $\hat{Y}_t = \phi_1 Y_{t-1} + \phi_2 Y_{t-2}$

Ŷ₆ = φ₁ × Y₅ + φ₂ × Y₄ = 0.65 × 114 + 0.30 × 110 = 74.10 + 33.00 = ₹107.10 ← Day 6 Prediction
4

Compute Residual Error

If the actual Day 6 price turns out to be ₹116, compute the residual and check if errors are random (white noise — ideal) or structured (model misspecification).

ε₆ = Actual − Forecast = 116 − 107.10 = +8.90 Residual analysis: → Plot residuals over time (should look random) → Ljung-Box test: H₀ = residuals are white noise (p > 0.05 = good) → Large systematic residuals → try higher p or ARIMA
5

Multi-Step Forecasting (Recursive)

To predict Day 7, use the Day 6 forecast (₹107.10) as Y_{t-1} and Day 5 actual (₹114) as Y_{t-2}. Each step uses previous forecasts — uncertainty grows with horizon.

Ŷ₇ = 0.65 × 107.10 + 0.30 × 114.00 = 69.615 + 34.20 = ₹103.82 Note: Forecast uncertainty ∝ horizon length 95% CI width expands as: ±1.96 × σ × √h (approximately)

AR Model Selection & Diagnostics Guide

✅ Use AR When...

  • PACF shows a clear cutoff at lag p
  • ACF decays exponentially or sinusoidally
  • Data is stationary (or made stationary)
  • Past values strongly predict future ones
  • You want an interpretable, parsimonious model

⚠️ Common Pitfalls

  • Choosing p too large → overfitting (use AIC/BIC)
  • Skipping stationarity check → invalid inference
  • Ignoring residual structure → try ARMA or ARIMA
  • Using AR for seasonal data without seasonal component
  • Extrapolating far beyond data range → uncertainty explodes

Moving Average Models — SMA, EMA & MA(q)

Simple MA (SMA)

Assigns equal weights to the last n observations. Easy to compute, smooth output, but reacts slowly — introduces a lag equal to (n-1)/2 periods.

Exponential MA (EMA)

Assigns geometrically decreasing weights. Recent data gets more influence. Faster reaction to trend changes — essential for trading signals like MACD.

MA(q) in ARIMA

The statistical MA(q) is different — it models the current value as a function of past forecast errors (residuals), not past values. Chosen via ACF cutoff.

Simple Moving Average (SMA)
$$SMA_t = \frac{1}{n}\sum_{i=0}^{n-1} Y_{t-i}$$
Exponential Moving Average (EMA)
$$EMA_t = \alpha \cdot Y_t + (1-\alpha)\cdot EMA_{t-1} \qquad \alpha = \frac{2}{n+1}$$
Statistical MA(q) — Used in ARIMA
$$Y_t = \mu + \varepsilon_t + \theta_1\varepsilon_{t-1} + \cdots + \theta_q\varepsilon_{t-q}$$
$\theta_i$ = MA coefficients  |  $\varepsilon_{t-i}$ = past forecast errors  |  $\mu$ = series mean. Note: MA(q) uses past errors, AR(p) uses past values.
⚖️
Theory: Weight Distribution Explained
1

SMA: Uniform Weights — A Rectangle

Every observation in the window gets weight 1/n. Data outside the window gets weight 0. This creates a rectangular (boxcar) filter in signal processing terms — it attenuates high-frequency fluctuations but preserves low-frequency trends.

SMA(3) weights: [1/3, 1/3, 1/3, 0, 0, ...] SMA(5) weights: [1/5, 1/5, 1/5, 1/5, 1/5, 0, ...] Lag = (n-1)/2 periods SMA(3) lags by 1 | SMA(5) lags by 2
2

EMA: Exponentially Decaying Weights

Expanding the EMA recursion reveals it's an infinite weighted average of all past observations, with weights decaying as $(1-\alpha)^k$. This gives EMA a long memory while still emphasising the recent past.

EMA_t = α·Yt + α(1-α)·Y_{t-1} + α(1-α)²·Y_{t-2}... α=0.50 (n=3): weights=[0.50, 0.25, 0.125, ...] α=0.33 (n=5): weights=[0.33, 0.22, 0.15, ...] Sum of all weights = 1 Effective memory ≈ 1/α periods
3

Choosing the Window Size n

The window size n controls the smoothing-responsiveness trade-off. Small n = jumpy but responsive. Large n = smooth but lagged. Use domain knowledge or cross-validation on a hold-out set.

Finance rule of thumb: Fast EMA (12-day) crosses above Slow EMA (26-day) → buy signal (MACD). For general forecasting: test n = 3, 5, 7, 10, 14 and pick lowest RMSE on validation set.
4

Statistical MA(q): Error Correction Model

Unlike SMA/EMA, the statistical MA(q) corrects for past forecast mistakes. If the model overpredicted last period (ε_{t-1} < 0), it adjusts this period's prediction downward by θ₁×ε_{t-1}.

ACF Rule for MA(q) order: • ACF sharp cutoff after lag q → use MA(q) • PACF decays slowly → supports MA model • ACF significant at lag 1 only → MA(1) • ACF significant at lags 1,2 → MA(2)

Worked Example: Temperature Smoothing & Forecast
Problem: Daily max temperatures (°C) for 7 days: 32, 35, 30, 33, 36, 34, 38. Apply SMA(3) and EMA(3) to smooth the data. Compare their accuracy. Forecast Day 8.
1

Compute α for EMA(3)

α = 2/(n+1) = 2/(3+1) = 0.50 → 50% weight to current, 50% to running EMA
2

Initialise EMA: Seed = First Observation

EMA₁ = Y₁ = 32°C (seed — no previous EMA)
3

Iterate EMA Day by Day

EMA₂ = 0.50×35 + 0.50×32.00 = 33.500 EMA₃ = 0.50×30 + 0.50×33.50 = 31.750 EMA₄ = 0.50×33 + 0.50×31.75 = 32.375 EMA₅ = 0.50×36 + 0.50×32.38 = 34.188 EMA₆ = 0.50×34 + 0.50×34.19 = 34.094 EMA₇ = 0.50×38 + 0.50×34.09 = 36.047
4

Compute SMA(3) — Starting from Day 3

SMA₃ = (32+35+30)/3 = 32.333 SMA₄ = (35+30+33)/3 = 32.667 SMA₅ = (30+33+36)/3 = 33.000 SMA₆ = (33+36+34)/3 = 34.333 SMA₇ = (36+34+38)/3 = 36.000
5

Forecast Day 8

EMA Day 8 = 0.50×38 + 0.50×36.047 = 37.02°C SMA Day 8 = (36+34+38)/3 = 36.00°C → EMA is more responsive to recent spike to 38°C

SMA vs EMA — Complete Comparison

SMA — When to Use

  • Long-term trend analysis without recent bias
  • Data has no dominant outliers
  • Equal-importance historical context needed
  • Baseline smoothing before ARIMA fitting
  • Economic data: unemployment, CPI, population

EMA — When to Use

  • High-frequency trading (MACD uses 12 & 26 day EMAs)
  • Volatile data where recency matters more
  • Reducing lag in trend following strategies
  • Live sensor data, IoT telemetry streams
  • When you need continuous smoothed values
Key Insight: ARIMA(0,1,1) is mathematically equivalent to Simple Exponential Smoothing (SES) — which is related to EMA with optimal α chosen by maximum likelihood. This bridges the gap between smoothing methods and the full ARIMA family.

ARIMA(p, d, q) — The Unified Model

p — AR Order

Number of lagged values used. Controls how much past levels influence the current value. Identified via PACF plot cutoff.

d — Differencing Order

Number of times the series is differenced to achieve stationarity. d=1 removes linear trend. d=2 removes quadratic trend.

q — MA Order

Number of lagged forecast errors used. Controls how much past shock propagation is modelled. Identified via ACF plot cutoff.

ARIMA — Backshift Operator Form
$$\underbrace{\Phi_p(B)}_{\text{AR}(p)}\;\underbrace{(1-B)^d}_{\text{Diff.}}\;Y_t = c + \underbrace{\Theta_q(B)}_{\text{MA}(q)}\;\varepsilon_t$$
Expanded ARIMA(1,1,1)
$$(1-\phi_1 B)(1-B)Y_t = (1 + \theta_1 B)\varepsilon_t$$
$B$ = backshift operator: $B \cdot Y_t = Y_{t-1}$  |  $\Delta Y_t = Y_t - Y_{t-1}$ = first difference
🧠
Theory: The ARIMA Construction Process
1

Step 1: Test Stationarity → Choose d

Run ADF test. If non-stationary, apply first difference ($\Delta Y_t = Y_t - Y_{t-1}$) and test again. Usually d=0 or d=1 suffices.

d=0: Series already stationary d=1: Linear trend removed by ΔYt = Yt − Y(t-1) d=2: Quadratic trend removed → Over-differencing introduces artificial MA structure
2

Step 2: Identify p and q from ACF/PACF

On the differenced series, examine the ACF and PACF plots:

ACF tail-off + PACF cuts at p → ARIMA(p,d,0) ACF cuts at q + PACF tail-off → ARIMA(0,d,q) Both tail off slowly → ARIMA(p,d,q) with p,q ≥ 1 Try multiple combos and compare AIC/BIC
3

Step 3: Estimate Parameters by MLE

ARIMA parameters are estimated by Maximum Likelihood Estimation (MLE), which maximises the probability of observing the data given the model.

Modern software (statsmodels, R's forecast) uses the Kalman filter to evaluate the exact log-likelihood efficiently, even for large time series.
4

Step 4: Diagnostic Checking

After fitting, verify the model is adequate by checking residuals:

✓ Ljung-Box: p > 0.05 → white noise residuals ✓ Shapiro-Wilk: p > 0.05 → normal residuals ✓ Residual ACF: all within ±1.96/√n bands ✓ Q-Q plot: points follow diagonal line If residuals show structure → increase p or q
5

Step 5: Forecast with Confidence Intervals

Point forecasts from ARIMA converge toward the series mean as the horizon grows. Confidence intervals widen because forecast uncertainty compounds at each future step.

1-step 95% CI: Ŷ(t+1) ± 1.96 × σ_ε h-step CI (ARIMA(1,0,0)): σ²(h) = σ²_ε × (1+φ₁²+φ₁⁴+...+φ₁²⁽ʰ⁻¹⁾) → CI expands with horizon h → ARIMA(1,1,0): CI expands as √h

Worked Example: Monthly Sales with ARIMA(1,1,1)
Problem: Monthly sales (units) for 6 months: 200, 220, 215, 235, 230, 250. Clear upward trend → non-stationary. Fit ARIMA(1,1,1) with φ₁ = 0.6, θ₁ = −0.4. Forecast Month 7.
1

Diagnose: Non-Stationary (d = 1 Needed)

Sales: [200, 220, 215, 235, 230, 250] Rolling mean: clearly increasing ↑ ADF p > 0.05 → non-stationary → apply d=1
2

First Difference — Remove Trend

ΔY₂ = 220−200 = +20 ΔY₃ = 215−220 = −5 ΔY₄ = 235−215 = +20 ΔY₅ = 230−235 = −5 ΔY₆ = 250−230 = +20 Series {+20,−5,+20,−5,+20} ← stationary ✓
3

Fit ARMA(1,1) on Differenced Series

Ẑ₇ = φ₁ × ΔY₆ + θ₁ × ε₆ = 0.6 × (+20) + (−0.4) × 0 = 12.00 + 0.00 = +12 units
4

Inverse Difference → Recover Forecast

Ŷ₇ = Y₆ + Ẑ₇ = 250 + 12 = 262 units ✓ 95% CI: 262 ± 1.96 × 16.2 ≈ [230.2, 293.8]
